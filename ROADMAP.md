# Roadmap

This roadmap is intentionally practical. Build the smallest useful assistant loop first, then add integrations once the shape is proven.

## Phase 1: Daily Planning Loop

Status: started.

- Create daily planning notes from a template.
- Capture most important thing, wife / relationship, calendar, follow-ups, finance/admin, and shutdown notes.
- Make carry-forward items explicit.
- Decide whether daily planning should produce a written note only or calendar blocks too.

## Phase 2: Memory Review Heartbeat

Status: not started.

- Scan recent daily notes and `memory/inbox/`.
- Append promotion candidates to `DREAMS.md`.
- Flag contradictions or stale memory.
- Ask before promoting sensitive or ambiguous facts.

## Phase 3: Obsidian Sync

Status: not started.

- Learn the vault structure.
- Decide which notes should be imported, linked, or mirrored.
- Build a cleanup workflow for stale notes and documents.

## Phase 4: Proton Calendar And Mail

Status: not started.

- Explore Proton integration options.
- Extract calendar commitments for daily planning.
- Extract email follow-ups and finance/admin items.
- Keep privacy and credential handling explicit.

## Phase 5: Finance Admin

Status: not started.

- Build a UK-focused finance/admin area.
- Track bills and subscriptions.
- Organize tax and investment/crypto notes.
- Flag practical optimization opportunities with appropriate caveats.

## Open Decisions

- Preferred daily planning time.
- Written daily plan only, calendar blocks, or both.
- Where Obsidian vault and finance documents live.
- How much automation should run without a direct prompt.
